\chapter{Experimental Setup and Results}
\label{chap:experiment}

This chapter describes the experimental setup used to evaluate the proposed methodology. The chapter first presents the benchmark environments and the computing infrastructure. Next, it specifies the experimental configuration, the baselines, and the performance evaluation metrics. Section~\ref{sec:empirical_results} then reports the experimental results across all three intrinsic reward families (count-based, ICM, and RIDE). Settings reported here are the standard configuration shared by every run. The full hyperparameter values are listed in Appendix~\ref{app:implementation}.

\section{Computing Environment}
\label{sec:computing_environment}

Experiments were executed on two complementary computing environments:
\begin{itemize}
    \item \textbf{Training (Google Colab, NVIDIA T4 GPU)}: the bulk of the $570$ training runs were dispatched to Google Colab notebooks running a single NVIDIA Tesla T4 ($16$\,GB VRAM) instance. Multiple Colab sessions were used in parallel so that several (environment, condition, seed) configurations could be trained concurrently within the available session quotas; each session saved its CSV reward log and \texttt{.pth}/\texttt{.meta.npz}/\texttt{.samples.npz} checkpoints back to local storage on completion.
    \item \textbf{Local development (AMD Ryzen 5 laptop, NVIDIA GTX 1650)}: code authoring, debugging, single-seed sanity checks, and figure/table generation (\texttt{plot\_compare.py} and \texttt{thesis/scripts/build\_results\_tables.py}) were run on a laptop with an AMD Ryzen 5 CPU and an NVIDIA GTX 1650 ($4$\,GB VRAM) GPU.
    \item \textbf{Operating system and software stack}: Windows 11 (laptop) / Linux (Colab); Python 3.12, PyTorch with CUDA support, \texttt{gymnasium} with the \texttt{minigrid} extension, \texttt{numpy}, \texttt{pandas}, and \texttt{matplotlib}.
    \item \textbf{Wall-clock budget}: each seeded run consists of $10^6$ environment steps; the wall-clock time per seed on the T4 ranged from $\sim 2$ hours for the no-intrinsic baseline to $\sim 4$ hours for the CARE variants, depending on the cost of the auxiliary networks (ICM and RIDE encoder/forward/inverse models, Beta Network).
\end{itemize}

The training scripts automatically dispatch the policy and auxiliary networks to GPU when CUDA is available and fall back to CPU otherwise. All experiments use deterministic random seeds for reproducibility, and all training logs and checkpoint files are saved to disk every $10^5$ steps.

\section{Experimental Configuration}
\label{sec:experimental_config}

All runs share the same PPO backbone and the same intrinsic-reward processing pipeline. This shared setup makes the comparison across modules and across scaling strategies fair. The full set of hyperparameter values is listed in Appendix~\ref{app:hyperparameters}. The most important high-level settings are summarized here.

No separate global intrinsic strength coefficient is used. The Beta Network output $\beta_\psi(s_t)$ alone sets the intrinsic-reward scale, and in fixed-$\beta$ conditions the same scalar $\beta$ plays this role. In the CARE conditions, the Beta Network is built as a two-layer encoder of dimension $256$ followed by a $128$-unit head producing $\log \beta_\psi$. This output is clipped so that $\beta_\psi(s_t) \in [\beta_\text{min}, \beta_\text{max}] = [10^{-4},\, 5\times 10^{-2}]$.

The range was chosen to cover the best fixed coefficients reported for sparse MiniGrid tasks: $\beta \approx 0.005$ for count-based bonuses and $\beta \approx 0.05$ for ICM \cite{andres2025impact}. The lower bound $\beta_\text{min} = 10^{-4}$ is set one order of magnitude below the smallest fixed value tested in this thesis ($\beta = 5 \times 10^{-4}$), giving the adaptive variant the freedom to go below the smallest tested fixed coefficient if doing so helps performance.

The advantage-correlation objective in Equation~\eqref{eq:beta_loss} uses regularization weight $\lambda_\text{reg} = 10^{-3}$ and reference value $\beta_0 = \sqrt{\beta_\text{min}\,\beta_\text{max}} \approx 2.24 \times 10^{-3}$ (the geometric mean of the bounds, the unique log-space midpoint of $[\beta_\text{min}, \beta_\text{max}]$). Gradient clipping is at $L_2$ norm $1.0$, and $L_2$ weight decay is $10^{-6}$ on the Beta Network parameters. The Beta Network is updated with Adam at learning rate $5 \times 10^{-4}$, with one $\psi$-step per outer PPO iteration. The live $\beta_\psi$ is used directly in the shaped reward (no Polyak target).

In the fixed-$\beta$ conditions, the Beta Network is replaced by a constant scalar $\beta \in \{0.0005,\, 0.001,\, 0.005,\, 0.01,\, 0.05\}$ and the correlation update is turned off. The rest of the pipeline (intrinsic generation, standardization, and the shaped reward formula in Equation~\eqref{eq:combined_reward}) is unchanged.

The base PPO algorithm uses $K = 80$ epochs per update, clip parameter $\epsilon = 0.2$, discount factor $\gamma = 0.99$, GAE bias-variance trade-off $\lambda = 0.95$, actor learning rate $3 \times 10^{-4}$, and critic learning rate $10^{-3}$. The rollout buffer length is $T = 4{,}000$ steps, and each seed is trained for $10^6$ environment steps. Each configuration is repeated over five independent random seeds $\{1, 2, 3, 4, 5\}$.

\section{Baselines}
\label{sec:baselines}

This thesis treats CARE as an adaptive layer that works with any intrinsic reward module. The three main intrinsic reward families used are: count-based bonuses, the Intrinsic Curiosity Module (ICM), and Rewarding Impact-Driven Exploration (RIDE). To separate the effect of CARE from the effect of the underlying intrinsic signal and from the choice of any particular fixed scaling coefficient, the baselines are organized into a $1 + 15 + 3$ matrix as summarized in Table~\ref{tab:baselines}: one no-intrinsic control, fifteen fixed-$\beta$ configurations covering a two-order-of-magnitude sweep across the three modules, and three CARE variants.

\begin{table}[H]
\centering
\caption{Baseline conditions evaluated on every environment.}
\label{tab:baselines}
\begin{tabular}{|m{1.8cm}|m{4.4cm}|m{7.2cm}|}
\hline
\textbf{Label} & \textbf{Configuration} & \textbf{Purpose in the comparison} \\ \hline
$B_0$ & PPO (no intrinsic reward) & Lower bound on exploration ability without any curiosity signal. \\ \hline
$F_{m,\beta}$ & PPO + module $m$ with constant $\beta$, $m \in \{\text{Count}, \text{ICM}, \text{RIDE}\}$, $\beta \in \{0.0005$, $0.001$, $0.005$, $0.01$, $0.05\}$ & Fixed-coefficient sweep for each module. The five values span two orders of magnitude and bracket the optimal coefficients reported for sparse MiniGrid tasks ($\beta \approx 0.005$ for count-based and $\beta \approx 0.05$ for ICM \cite{andres2025impact}). \\ \hline
$C_\text{Count}$ & PPO + count-based + CARE & Count-based bonus with learned $\beta_\psi(s_t)$ from the Beta Network. \\ \hline
$C_\text{ICM}$ & PPO + ICM + CARE & ICM bonus with learned $\beta_\psi(s_t)$. \\ \hline
$C_\text{RIDE}$ & PPO + RIDE + CARE & RIDE bonus with learned $\beta_\psi(s_t)$. \\ \hline
\end{tabular}
\end{table}

This design isolates three distinct effects:
\begin{itemize}
    \item \textbf{$B_0$ vs $F_{m,\beta}$} measures whether intrinsic motivation in module $m$ at coefficient $\beta$ helps or hurts on top of the extrinsic-only PPO baseline. Sweeping $\beta$ over two orders of magnitude around the values reported in the literature shows how sensitive each module is to manual tuning.
    \item \textbf{$F_{m,\beta}$ vs $C_m$} measures the effect of CARE's adaptive scaling, with the underlying intrinsic signal held fixed. The clearest test is the comparison of $C_m$ against the \emph{best} fixed-$\beta$ value for module $m$ on each environment. If CARE matches or beats the best post-hoc fixed coefficient \emph{without} having to search for that coefficient, the case for state-dependent scaling is established.
    \item \textbf{$C_\text{Count}$ vs $C_\text{ICM}$ vs $C_\text{RIDE}$} shows how the same CARE layer behaves with intrinsic generators of different structure (count-based visitation, ICM forward error, and RIDE impact-with-episodic-counts). This answers the research question of module-agnosticism stated in Chapter~1.
\end{itemize}

With $1 + 15 + 3 = 19$ conditions per environment and five seeds per condition, the full evaluation comprises $19 \times 5 = 95$ runs per environment and $95 \times 6 = 570$ runs across the six environments.

\section{Performance Evaluation Metrics}
\label{sec:metrics}

Four metrics are used to quantify and compare the conditions defined above.

\textbf{Episode return.} Let $R^E_t$ be the extrinsic reward at time $t$ in evaluation episode $e$. The episode return for that episode is $\mathcal{R}_e = \sum_t R^E_t$. The reported curves are the mean of $\mathcal{R}_e$ over a sliding window of evaluation episodes around each checkpoint, averaged across the five seeds. Episode return is the primary metric of task success.

\textbf{Sample efficiency.} For each environment, a target return $r^\star$ is set at a fraction (usually $0.7$) of the final return achieved by the strongest baseline. The sample efficiency of a method is then the smallest number of environment steps $n^\star$ at which its mean evaluation return first reaches $r^\star$:
\begin{equation*}
n^\star \;=\; \min\bigl\{ n : \overline{\mathcal{R}}(n) \ge r^\star \bigr\},
\end{equation*}
with $n^\star = +\infty$ if the threshold is never reached within the $10^6$-step budget. Lower values of $n^\star$ indicate faster learning.

\textbf{Learning stability.} For each method and each timestep $n$, the learning stability is reported as the across-seed standard deviation $\sigma(n)$ of the mean evaluation return at that timestep:
\begin{equation*}
\sigma(n) \;=\; \sqrt{\frac{1}{|S| - 1} \sum_{s \in S} \bigl( \overline{\mathcal{R}}_s(n) - \overline{\mathcal{R}}(n) \bigr)^2},
\end{equation*}
where $S$ is the set of seeds. Lower $\sigma(n)$ means more reproducible behavior.

\textbf{Learned scaling behavior.} The learned function $\beta_\psi$ is analyzed at both the global and local level. Globally, the distribution of $\beta_\psi(s_t)$ over the rollout buffer is summarized by its mean and selected percentiles ($25\%, 50\%, 75\%$) at fixed checkpoints during training. Locally, per-environment heatmaps of $\beta_\psi(s)$ projected onto the agent's two-dimensional position are produced from trajectory data saved every $10^5$ steps. These heatmaps give a visual view of where in state space CARE puts more exploration influence.

\section{Empirical Results}
\label{sec:empirical_results}

This section reports the experimental evaluation of all $19$ conditions defined in Section~\ref{sec:baselines} across the six MiniGrid environments described in Section~\ref{sec:benchmark_environments}. Each condition is run with five seeds, for a total of $570$ training runs of $10^6$ steps each. The presentation is built around three research questions posed in Chapter~\ref{ch:methodology}. First, does CARE match or beat the best post-hoc fixed coefficient on a per-(environment, module) basis (Section~\ref{sec:results_module_level})? Second, how does the same CARE layer behave with intrinsic generators of different structure (Section~\ref{sec:results_cross_module})? Third, how sensitive is each module to the choice of $\beta$, which calibrates the expected benefit of state-dependent scaling (Section~\ref{sec:results_sensitivity})? Section~\ref{sec:results_beta_behavior} then examines the learned $\beta_\psi(s)$ in a more visual way. Section~\ref{sec:results_discussion} discusses the findings in light of the formal results from Chapter~\ref{ch:methodology}.

\textbf{Count-based.} The count-based bonus is the most well-behaved of the three modules: CARE-COUNT either matches or comes very close to the best fixed coefficient on every environment with a usable extrinsic signal, and the losses are confined to two tasks where curiosity needs to be actively suppressed late in the episode. Stepping through the six environments in turn:

\begin{itemize}
    \item \textbf{DoorKey-8x8}: CARE-COUNT matches the best fixed coefficient ($0.973 \pm 0.001$ vs $0.973 \pm 0.001$).
    \item \textbf{Empty-16x16}: CARE-COUNT trails by $-0.017$ ($0.959 \pm 0.028$ vs $0.975 \pm 0.000$), but this reflects a speed/asymptote trade-off rather than a regression. Table~\ref{tab:sample_efficiency_count} shows that CARE-COUNT reaches the $0.7$-of-best threshold faster than every fixed coefficient on this environment ($66\text{k}$ vs $82\text{k}$ steps), while two of its five seeds plateau slightly below the optimum and raise the across-seed standard deviation to $\pm 0.028$. This is consistent with the cold-start regime from Corollary~\ref{cor:degeneration}(i): in Empty-16x16, the extrinsic GAE advantage carries little cross-state signal, so the correlation gradient is noise-dominated and $\beta_\psi$ stays near $\beta_0$, behaving like a small but non-zero fixed coefficient.
    \item \textbf{KeyCorridor}: CARE-COUNT edges out the best fixed coefficient by $+0.003$ ($0.831 \pm 0.068$ vs $0.828 \pm 0.103$) and has roughly half the across-seed standard deviation.
    \item \textbf{LavaCrossing-S9N3}: CARE-COUNT trails by $-0.158$ ($0.372 \pm 0.317$ vs $0.530 \pm 0.305$ for $\beta = 0.001$). The within-batch correlation gradient pushes $\beta$ above the level a smaller global fixed $\beta$ would set, which hurts when curiosity needs to be suppressed near lava terminations.
    \item \textbf{RedBlueDoors-8x8}: CARE-COUNT trails by $-0.126$ ($0.653 \pm 0.269$ vs $0.779 \pm 0.047$ for $\beta = 0.0005$). The failure mode is the same as on LavaCrossing: curiosity should drop sharply once the second door becomes the next subgoal.
    \item \textbf{UnlockPickup}: CARE-COUNT matches the best fixed coefficient ($0.936 \pm 0.002$ vs $0.936 \pm 0.006$).
\end{itemize}

\noindent CARE-COUNT also has the lowest variance on every environment where the comparison converges and the agent has a non-trivial extrinsic signal. This matches the idea that state-dependent scaling tracks the per-state extrinsic advantage signal across seeds.

\textbf{ICM.} ICM is the most fragile of the three modules with respect to the choice of $\beta$, and CARE inherits part of that fragility. On environments where the small fixed values $\{0.0005,\,0.001,\,0.005\}$ all converge, CARE-ICM displays large seed-to-seed variance, and on environments where the ICM forward error stays high off the goal path the correlation objective amplifies the wrong states. Stepping through the six environments in turn:

\begin{itemize}
    \item \textbf{DoorKey-8x8}: CARE-ICM reaches $0.770 \pm 0.340$ vs $0.973 \pm 0.001$ for the best fixed coefficient ($\beta = 0.0005$), with the gap driven mostly by a single failure seed rather than systematic underperformance.
    \item \textbf{Empty-16x16}: CARE-ICM reaches $0.575 \pm 0.525$ vs $0.975 \pm 0.000$ for the best fixed coefficient ($\beta = 0.0005$). The very large variance again indicates a small number of failure seeds rather than a uniformly degraded policy.
    \item \textbf{KeyCorridor}: the clearest failure mode in the entire study. CARE-ICM falls to $0.000 \pm 0.000$ across all five seeds, while the best fixed coefficient ($\beta = 0.001$) reaches $0.457 \pm 0.312$. On this multi-room layout the ICM forward error stays large in regions far from the goal; the correlation gradient of $L_\text{corr}$ keeps pushing $\beta_\psi$ up in those regions throughout training, which prevents the agent from ever moving from exploration to exploitation.
    \item \textbf{LavaCrossing-S9N3}: CARE-ICM edges out the best fixed coefficient by $+0.017$ ($0.077 \pm 0.165$ vs $0.060 \pm 0.134$ for $\beta = 0.001$), but the gain is small and well within the seed-level noise band.
    \item \textbf{RedBlueDoors-8x8}: CARE-ICM reaches $0.273 \pm 0.350$ vs $0.737 \pm 0.071$ for the best fixed coefficient ($\beta = 0.0005$), a large $-0.464$ loss for the same reason as on KeyCorridor: the forward-error signal is large in regions that are not on the path to the second door.
    \item \textbf{UnlockPickup}: CARE-ICM reaches $0.716 \pm 0.315$ vs $0.937 \pm 0.002$ for the best fixed coefficient ($\beta = 0.001$), again driven by a single failure seed.
\end{itemize}

\noindent The pattern across the six cells matches the design limitation flagged in Section~\ref{sec:progress_loss}: when the underlying intrinsic signal is large in regions that are not goal-directed, the correlation objective amplifies $\beta_\psi$ in exactly those regions.

\subsection{Effect of CARE Across Modules}
\label{sec:results_cross_module}

Keeping the underlying intrinsic signal fixed and changing only the scaling strategy isolates the effect of CARE itself. Table~\ref{tab:care_vs_best_fixed} summarizes the per-(environment, module) comparison between $C_m$ and the best post-hoc fixed-$\beta$ value for the same module. Across the eighteen comparisons (six environments times three modules), CARE matches or beats the best fixed coefficient in $7$ of $18$ cells, of which $3$ are clear wins with gap above $+0.005$ raw reward (LavaCrossing-ICM, LavaCrossing-RIDE, and UnlockPickup-RIDE) and $4$ are within $\pm 0.005$ of the best fixed value (DoorKey-COUNT, DoorKey-RIDE, KeyCorridor-COUNT, and UnlockPickup-COUNT). CARE trails in the remaining $11$ cells. The wins concentrate in the count-based and RIDE pairings, and the losses concentrate in the ICM pairings.

CARE-ICM clusters with the worst fixed-ICM coefficients. The reading, in line with Proposition~\ref{prop:target_recovery}, is that CARE's adaptive scaling is most useful when the underlying intrinsic generator already provides a signal whose size correlates with future extrinsic advantage at the state level. Count-based bonuses are large at unvisited states. On MiniGrid, these states match real progress along the task graph. RIDE bonuses are large at high-impact transitions. On these environments, those transitions match key pick-ups and door openings. ICM bonuses, in contrast, are large wherever the forward model is currently inaccurate. That inaccuracy pattern can stay in regions of the state space that are not on the path to extrinsic reward. This is exactly the failure mode visible on KeyCorridor.

\subsection{Sensitivity to Fixed \texorpdfstring{$\beta$}{beta}}
\label{sec:results_sensitivity}

Figures~\ref{fig:all_envs_count}--\ref{fig:all_envs_ride} also show how sensitive each module is to the fixed scaling coefficient. The five-value sweep $\beta \in \{0.0005,\,0.001,\,0.005,\,0.01,\,0.05\}$ spans two orders of magnitude. Across all eighteen (environment, module) cells, the largest value $\beta = 0.05$ is a clear failure. It drives final reward to within noise of zero on every cell, even on environments like Empty-16x16 where every smaller value converges to near-optimal performance. This is the experimental reason for clipping $\beta_\text{max} = 5 \times 10^{-2}$ in the Beta Network. A larger upper bound would push CARE into a regime where the intrinsic signal overwhelms the extrinsic objective and the policy breaks down.

The pattern of best fixed-$\beta$ values in Table~\ref{tab:care_vs_best_fixed} is also informative. The best post-hoc value lies at $\beta = 0.0005$ (the smallest swept value) in $9$ of $18$ cells, and at $\beta = 0.001$ in $7$ of $18$. Only $2$ cells prefer $\beta = 0.005$ or larger. This concentration near the lower end of the sweep is itself an argument for adaptive scaling. A practitioner who tuned $\beta$ on DoorKey-8x8 with a coarse grid that includes $\beta = 0.005$ would land on a value $10\times$ too large for KeyCorridor with the same module. CARE-COUNT and CARE-RIDE handle this kind of cross-environment mismatch by adapting per state. This is why their environment-by-environment trace in Figures~\ref{fig:all_envs_count}--\ref{fig:all_envs_ride} stays inside the band of the best two fixed coefficients on most environments without any environment-specific search.

The two-orders-of-magnitude sweep also shows that the modules differ in how sharp their sensitivity is. The count-based module degrades smoothly from $\beta = 0.0005$ to $\beta = 0.01$ on most environments, with a cliff at $\beta = 0.05$. ICM is sharper: on KeyCorridor, the gap between $\beta = 0.001$ ($0.457$) and $\beta = 0.005$ ($0.064$) is already an order of magnitude in performance. This is the case even though both values are below the best $\beta$ reported in the literature for ICM on similar tasks. RIDE sits in between: a wider plateau than ICM but a sharper drop than count-based. CARE's best gains in this study (LavaCrossing for count-based and RIDE, UnlockPickup for RIDE) come on environments that lie in the sharper regions of these sensitivity curves. These are the regions where state-dependent adaptation has the most room to help.

\subsection{Discussion}
\label{sec:results_discussion}

The experimental findings can be read against the formal results from Chapter~\ref{ch:methodology} in three ways.

First, the cold-start regime described by Corollary~\ref{cor:degeneration}(i) is observed in practice. On the three environments where extrinsic reward only appears mid-training (KeyCorridor, RedBlueDoors, UnlockPickup), the mean $\beta_\psi$ stays close to the prior $\beta_0$ during the early phase. The correlation gradient only takes over once the extrinsic advantage gets a non-trivial size. This supports the design choice of placing the regularizer's reference value at the geometric mean of the bounds. This choice anchors the cold-start behavior at a value that is neither too exploratory nor close to zero. The correlation objective can then push $\beta_\psi$ in either direction once it has signal to work with.

Second, the cross-moment direction described by Proposition~\ref{prop:target_recovery} maps to the observed module-by-module pattern. On modules where the intrinsic signal $I^{+,m}$ tends to match the future extrinsic advantage at the state level (count-based, where unvisited states sit on the path to the goal, and RIDE, where high-impact transitions sit at the task-relevant decision points), the within-batch covariance is positive on average. CARE then raises $\beta_\psi$ at exactly the states where raising it is helpful. On ICM, the intrinsic signal is driven by forward-model error that does not always match the extrinsic-success path. The conditional covariance can be positive in regions that are not goal-directed. The same gradient then raises $\beta_\psi$ where raising it harms the policy. The KeyCorridor failure of CARE-ICM is the clearest experimental case of this mismatch.

Third, in light of the literature reviewed in Chapter~\ref{ch:literature}, the comparison can be read as a partial answer to the module-agnosticism question raised by Andres et al.\ \cite{andres2025impact}. On modules whose raw signal already correlates with extrinsic advantage at the state level, CARE matches or beats the post-hoc-best fixed coefficient without any environment-specific search. On modules whose raw signal does not, CARE inherits the mismatch and can make it worse. This matches the view that CARE is not a replacement for a well-chosen intrinsic generator. CARE is a state-dependent multiplier on top of one. Because of the correlation objective, the multiplier can only be as informative as the correlation between the underlying signal and extrinsic learning slack. The practical message is that CARE is a safe default with count-based or impact-driven bonuses on sparse-reward gridworld tasks. CARE should be used with care when paired with curiosity-by-prediction-error bonuses such as ICM in the same setting.

The main limit of the present study is the gridworld scope. MiniGrid environments share a discrete action space, low-dimensional observations, and a reward structure in which extrinsic feedback appears within the $10^6$-step budget. In higher-dimensional or richer-observation environments, the cold-start regime is expected to last longer. The correlation objective would have to stay well-conditioned for many more updates before extrinsic signal arrives. Whether the picture from this study (CARE most useful with count-based and impact-driven generators, least useful with prediction-error ones) carries over to those settings is the natural follow-up direction.
