\chapter{Methodology}
\label{ch:methodology}

This chapter presents the proposed methodology in formal terms. The chapter first defines the overall system architecture on top of the intrinsic-augmented decision process. Next, it describes the intrinsic reward generator used in the implemented system. The chapter then builds the Correlation-Aware Reward Exploration (CARE) framework with its Beta Network and learning-progress objective. The last section gives the full training algorithm. Five formal results (Propositions 3.1--3.3, Lemma 3.1, Corollary 3.1) are stated in this chapter and proved in Appendix A.

\section{System Architecture}
\label{sec:system_architecture}

The proposed framework is built on top of the intrinsic-augmented MDP (Definition~\ref{def:augmented_mdp} in Chapter~\ref{ch:literature}) and introduces a learned, state-dependent scaling function for intrinsic rewards. The shaping function is formalized in the following definition.

\begin{definition}[State-Dependent Reward Shaping]
\label{def:reward_shaping}
A \emph{state-dependent reward shaping function} is a measurable map $\beta : \mathcal{S} \to \mathbb{R}_+$. Given an intrinsic-augmented MDP, the corresponding shaped reward at time step $t$ is
\begin{equation}
\bar{r}_t = R_t^E + \beta(s_t)\, R_t^I.
\label{eq:shaped_reward}
\end{equation}
The standard fixed-coefficient formulation corresponds to the special case $\beta(s) \equiv \beta_0$ for some constant $\beta_0 > 0$.
\end{definition}

The proposed system uses this form with a parameterized $\beta_\psi$ together with one of three intrinsic reward generators: count-based exploration, the Intrinsic Curiosity Module, or Rewarding Impact-Driven Exploration. All three are evaluated in Chapter~\ref{chap:experiment}. The full pipeline is shown in Figure~\ref{fig:care_arch}.

\begin{figure}[H]
\centering
\includegraphics[width=\linewidth]{figures/system_architecture.png}
\caption{CARE system architecture. The intrinsic module block stands for any of the three implementations evaluated in this thesis (count-based, ICM, or RIDE); the rest of the pipeline is identical across modules. Solid arrows trace the forward computation that produces the shaped reward $\bar r_t$ used by PPO. Dashed arrows indicate the auxiliary signals used to update the Beta Network $\beta_\psi$ via the correlation objective $L_\beta$, which couples the weighted intrinsic reward $\beta_\psi\, I_t^{+,m}$ to the extrinsic GAE advantage $\hat A_t^{E}$ obtained from the critic.}
\label{fig:care_arch}
\end{figure}

The shaped reward used throughout this thesis follows directly from Definition~\ref{def:reward_shaping}. For the chosen intrinsic reward module $m$ with rectified intrinsic signal $I_t^{+,m}$ (Section~\ref{sec:standardization}), the combined reward optimized by PPO is
\begin{equation}
\bar{r}_t \;=\; R_t^E \;+\; \beta_\psi(s_t)\, I_t^{+,m}.
\label{eq:combined_reward}
\end{equation}
In simple terms, $\beta_\psi(s_t)$ acts as a state-dependent volume control on curiosity. The Beta Network raises intrinsic reward in states where exploration goes together with future extrinsic success, and lowers it elsewhere. The rest of this chapter makes this idea formal.

\section{Intrinsic Reward Generation}
\label{sec:intrinsic_modules}

The proposed framework supports three intrinsic reward modules: count-based exploration \cite{bellemare2016count, tang2017exploration}, the Intrinsic Curiosity Module (ICM) \cite{pathak2017curiosity}, and Rewarding Impact-Driven Exploration (RIDE) \cite{raileanu2020ride}. All three modules differ only in how the raw intrinsic signal $I_t$ is computed at each transition. The rest of the pipeline (standardization, Beta Network, and learning-progress objective) is the same. Chapter~\ref{chap:experiment} reports the results for all three modules. The rest of this section describes each module with its raw-reward formula. The last subsection covers the shared standardization step that produces the rectified signal $I_t^+$ fed to the Beta Network.

\subsection{ICM Instantiation}
\label{sec:icm}

\textbf{Forward and Inverse Dynamics Models.} ICM first encodes each state into a latent representation $\phi(s_t) \in \mathbb{R}^d$. ICM then trains two extra models. The forward dynamics model $f^F_\eta$ predicts the next latent feature from the current feature and action. The inverse dynamics model $f^I_\eta$ predicts the action from two consecutive latent states. The combined ICM loss is
\begin{equation}
L_\text{ICM}(\eta) \;=\;
\alpha_F \, \mathbb{E}\!\left[\tfrac{1}{2}\!\left\| f^F_\eta(\phi_t, a_t) - \phi_{t+1} \right\|_2^2\right]
\;+\; \alpha_I \, \mathbb{E}\!\left[ -\log p_\eta(a_t \mid \phi_t, \phi_{t+1}) \right],
\label{eq:icm_loss}
\end{equation}
with weighting coefficients $\alpha_F, \alpha_I > 0$.

\textbf{Raw Intrinsic Reward.} The raw intrinsic reward at time step $t$ is the forward prediction error
\begin{equation}
I_t \;=\; \tfrac{1}{2}\!\left\| f^F_\eta(\phi_t, a_t) - \phi_{t+1} \right\|_2^2.
\label{eq:icm_raw}
\end{equation}

\subsection{Count-Based Exploration}
\label{sec:count_module}

In the count-based instantiation, the raw intrinsic reward at state $s_t$ is a function of the empirical visit count of that state. Following \cite{tang2017exploration}, we use a random-projection hash $\phi_h : \mathcal{S} \to \mathbb{Z}^{d_h}$ that maps each state to a discrete hash code:
\begin{equation}
\phi_h(s_t) \;=\; \mathrm{round}\!\left( \frac{W^\top s_t}{\Delta} \right),
\label{eq:count_hash}
\end{equation}
where $W \in \mathbb{R}^{d_s \times d_h}$ is a random projection matrix sampled once at initialization, $d_h$ is the hash dimension, and $\Delta > 0$ is the bin width. Let $N(\phi_h(s_t))$ denote the number of times the hash code $\phi_h(s_t)$ has been visited prior to time step $t$. The raw intrinsic reward is the inverse-square-root visitation bonus
\begin{equation}
I_t \;=\; \frac{1}{\sqrt{\max\{N(\phi_h(s_t)),\, 1\}}}.
\label{eq:count_raw}
\end{equation}
This module has no learnable parameters; the hash table $N(\cdot)$ is updated incrementally as new states are visited.

\subsection{Rewarding Impact-Driven Exploration}
\label{sec:ride_module}

In the RIDE design \cite{raileanu2020ride}, the raw intrinsic reward measures the size of representation change between consecutive states in a learned embedding space, divided by an episodic visitation count. Like ICM, RIDE learns an encoder $\phi$ and extra forward and inverse dynamics models $f^F_\eta, f^I_\eta$. These models are trained with the same combined loss $L_\text{ICM}(\eta)$ from Equation~\eqref{eq:icm_loss}. Unlike ICM, the forward prediction error is used \emph{only} to train the encoder, never as the intrinsic signal. Instead, the raw intrinsic reward is the impact of the transition on the latent representation, divided by an episodic count:
\begin{equation}
y_t \;=\; \phi(s_t) \in \mathbb{R}^{d_y},
\label{eq:ride_embed}
\end{equation}
\begin{equation}
I_t \;=\; \frac{\bigl\| \phi(s_{t+1}) - \phi(s_t) \bigr\|_2}{\sqrt{\max\{ N_e(\phi_h(s_{t+1})),\, 1 \}}},
\label{eq:ride_raw}
\end{equation}
where $\phi_h$ is the same random-projection hash defined in Equation~\eqref{eq:count_hash} and $N_e(\cdot)$ counts the number of visits to the corresponding hash code \emph{within the current episode} (the count is reset at every episode boundary). The numerator rewards transitions whose representation change is large. The idea is that useful actions move the agent through latent space, not leave it in place. The denominator reduces bonuses for revisiting states already seen in the same episode. This prevents the cyclical wandering that happens when prediction-based bonuses are used in procedurally generated environments. Because the encoder is updated by the same inverse-plus-forward objective as ICM, RIDE inherits ICM's representation learning. But it replaces the intrinsic signal with an episodic, impact-driven one. We use the same hash dimension $d_h = 32$ and bin width $\Delta = 0.1$ as in the count-based module, and an embedding dimension of $d_y = 256$ matching the ICM encoder.

\subsection{Standardization and Rectification}
\label{sec:standardization}

No matter which module is used, the raw intrinsic reward $I_t$ from Equation~\eqref{eq:icm_raw}, \eqref{eq:count_raw}, or \eqref{eq:ride_raw} is non-stationary. Its size changes during training as the underlying generator (forward model, hash counts, or learned encoder) adapts to the agent's experience. To keep the input to the Beta Network bounded and roughly stable across updates, the same preprocessing step is applied to all three modules.

\begin{definition}[Standardized Intrinsic Reward]
\label{def:standardized_intrinsic}
Given a minibatch with empirical mean $\mu_I$ and standard deviation $\sigma_I$ of the raw intrinsic rewards, and a small constant $\varepsilon > 0$, the \emph{standardized intrinsic reward} is
\begin{equation}
I_t^+ \;=\; \max\!\left( 0,\; \frac{I_t - \mu_I}{\sigma_I + \varepsilon} \right).
\label{eq:icm_standardized}
\end{equation}
\end{definition}

This preprocessing throws away transitions whose intrinsic signal is below the batch average, and keeps the more surprising transitions. We use the general notation $I_t^{+,m}$ to make clear that the same pipeline (standardization, Beta Network, correlation objective) is applied the same way across all three modules. Only the raw signal $I_t$ differs.

\section{Correlation-Aware Reward Exploration}
\label{sec:care}

CARE adds a state-dependent scaling function $\beta_\psi(s_t)$ between the intrinsic reward generator and the policy update. This section defines the network, its advantage-correlation training objective, and why we choose a first-order optimization scheme over a meta-gradient one.

\subsection{Beta Network}
\label{sec:beta_network}

\begin{definition}[Beta Network]
\label{def:beta_network}
The \emph{Beta Network} is a parameterized map
\begin{equation}
\beta_\psi : \mathcal{S} \to [\beta_\text{min}, \beta_\text{max}], \qquad 0 < \beta_\text{min} < \beta_\text{max} < \infty,
\label{eq:beta_network}
\end{equation}
which assigns a positive, bounded scaling factor to each state. In the implemented system, $\beta_\psi$ is realized as a feed-forward MLP with a 2-layer encoder of width $256$ and a 2-layer head of widths $[256, 128, 1]$ that produces $\log \beta_\psi(s_t)$. The output is mapped to $[\beta_\text{min}, \beta_\text{max}] = [10^{-4},\, 5\times 10^{-2}]$ via $\beta_\psi(s_t) = \exp\bigl( \mathrm{clamp}(\log \beta_\psi(s_t),\, \log \beta_\text{min},\, \log \beta_\text{max}) \bigr)$. The reference value used by the regularizer in Section~\ref{sec:progress_loss} is the geometric mean $\beta_0 = \sqrt{\beta_\text{min}\,\beta_\text{max}} \approx 2.24 \times 10^{-3}$; this choice places $\log \beta_0$ at the midpoint of $[\log \beta_\text{min},\, \log \beta_\text{max}]$, so the prior is symmetric in log space. The complete architecture and weight initialization are listed in Appendix~\ref{app:architectures}, Table~\ref{tab:beta_arch}.
\end{definition}

For an intrinsic reward module $m$, the scaled intrinsic reward at time step $t$ is
\begin{equation}
\tilde I_t^{\,m} \;=\; \beta_\psi(s_t)\, I_t^{+,m},
\label{eq:scaled_intrinsic}
\end{equation}
and substituting Equation~\eqref{eq:scaled_intrinsic} into Equation~\eqref{eq:combined_reward} yields the shaped reward used by PPO.

The bounded range built into Definition~\ref{def:beta_network} makes sure that the shaped reward cannot grow without limit relative to the extrinsic signal, no matter how the Beta Network behaves during training.

\begin{lemma}[Bounded Shaped Reward]
\label{lem:bounded_shaped}
Let $\beta_\psi$ satisfy Definition~\ref{def:beta_network} and assume the standardized intrinsic reward is bounded almost surely, $|I_t^{+,m}| \le M$ for some constant $M < \infty$. Then for every time step $t$,
\begin{equation*}
\left| \bar{r}_t - R_t^E \right| \;\le\; \beta_\text{max}\, M.
\end{equation*}
Proof in Appendix~\ref{app:proof_bounded}.
\end{lemma}

Lemma~\ref{lem:bounded_shaped} is what lets CARE be inserted into a PPO training loop without adding instability. The shaped reward differs from the extrinsic reward by a bounded amount. So the policy gradient stays bounded under the same smoothness conditions that PPO already needs.

\subsection{Advantage-Correlation Objective}
\label{sec:progress_loss}

The Beta Network is trained so that the weighted intrinsic reward $\beta_\psi(s_t)\, I_t^{+,m}$ is large at states where the extrinsic critic shows that the policy is doing better than its own baseline, and small where it is not. The signal that captures this idea of ``where extrinsic learning still has room to grow'' is the extrinsic GAE advantage $\hat A_t^{E}$. This signal is already computed by the PPO pipeline. Linking intrinsic curiosity to extrinsic advantage through a within-batch correlation objective gives a first-order training rule for $\psi$. This rule needs no extra rollouts and no differentiation through the policy update.

For each transition in the rollout buffer, the GAE \cite{schulman2016gae} advantage with respect to the extrinsic value function $V_\mu$ is
\begin{equation}
\hat A_t^{E} \;=\; \sum_{l=0}^{T-1-t} (\gamma\,\lambda)^l\, \delta_{t+l}^{E},
\qquad
\delta_t^{E} \;=\; R_t^{E} \;+\; \gamma\, V_\mu(s_{t+1})\,(1 - d_t) \;-\; V_\mu(s_t),
\label{eq:gae_advantage_ext}
\end{equation}
with discount factor $\gamma$, GAE bias-variance trade-off $\lambda$, and episode termination flag $d_t \in \{0,1\}$. The advantages used to supervise $\psi$ are computed without rollout-level normalization. This keeps the cross-state size of $\hat A_t^{E}$ intact. That size carries the signal-strength information needed to weight intrinsic curiosity.

The Beta Network is trained on minibatches of size $B$ drawn from the rollout buffer. Within each minibatch, both the weighted intrinsic reward and the extrinsic advantage are standardized:
\begin{equation}
\tilde I_t \;=\; \beta_\psi(s_t)\, I_t^{+,m},
\qquad
\hat I_t^{z} \;=\; \frac{\tilde I_t - \mu_{\tilde I}}{\sigma_{\tilde I} + \varepsilon},
\qquad
\hat A_t^{z} \;=\; \frac{\hat A_t^{E} - \mu_{\hat A^E}}{\sigma_{\hat A^E} + \varepsilon},
\label{eq:zscored_signals}
\end{equation}
where $\mu_\bullet, \sigma_\bullet$ denote the empirical mean and (biased) standard deviation over the minibatch, and $\varepsilon > 0$ is a numerical-stability constant. The correlation loss is the negative within-batch sample correlation of $\tilde I_t$ and $\hat A_t^{E}$:
\begin{equation}
L_\text{corr}(\psi)
\;=\;
-\,\frac{1}{B}\sum_{t=1}^{B}\, \hat I_t^{z}\,\hat A_t^{z}.
\label{eq:corr_loss}
\end{equation}
Minimizing $L_\text{corr}$ is equivalent to maximizing the Pearson correlation between $\tilde I$ and $\hat A^{E}$ over the minibatch, since the z-scored variables have unit variance by construction.

A log-space regularizer pulls $\beta_\psi$ toward the reference value $\beta_0 = \sqrt{\beta_\text{min}\,\beta_\text{max}}$ to prevent degenerate scaling and to control variance during early training:
\begin{equation}
L_\text{reg}(\psi) \;=\; \mathbb{E}\!\left[ \bigl( \log \beta_\psi(s_t) - \log \beta_0 \bigr)^2 \right].
\label{eq:reg_loss}
\end{equation}
The complete Beta-network objective is
\begin{equation}
L_\beta(\psi) \;=\; L_\text{corr}(\psi) \;+\; \lambda_\text{reg}\, L_\text{reg}(\psi),
\label{eq:beta_loss}
\end{equation}
with $\lambda_\text{reg} > 0$ the regularization weight. No separate weighting on $L_\text{corr}$ is needed because z-scoring fixes its scale to $O(1)$ regardless of the underlying magnitudes of $\tilde I$ and $\hat A^{E}$.

The use of \emph{advantage} rather than the discounted return $G_t^{E}$ \cite{andres2025impact} is on purpose. The variance of $G_t^{E}$ is mainly caused by episode position. Early-episode states see almost the full $\sum_{k} \gamma^{k} R_{t+k}^{E}$, while late-episode states see almost none. So a correlation between $\tilde I_t$ and $G_t^{E}$ is mostly a correlation between $\tilde I_t$ and ``time left in the episode''. Subtracting the value baseline $V_\mu(s_t)$, as GAE does, removes this confound. The result is a per-state credit signal: $\hat A_t^{E}$ is positive at states where the realized extrinsic return beats the critic's prediction, and negative where it falls short. Linking $\beta_\psi$ to this signal therefore raises intrinsic curiosity at states where extrinsic learning is still moving upward, and lowers it elsewhere.

The correlation objective has a clean statistical interpretation, and its gradient has a clean directional one.

\begin{proposition}[Optimal State-Dependent Scaling]
\label{prop:target_recovery}
Assume the Beta Network has sufficient expressive capacity to represent any measurable function $\mathcal{S} \to [\beta_\text{min}, \beta_\text{max}]$, and that within the rollout buffer the standardized advantage $\hat A_t^{z}$ has nonzero conditional cross-moment with the standardized intrinsic signal at almost every state. In the population limit and with $\lambda_\text{reg} = 0$, every minimizer $\psi^\star$ of $L_\beta(\psi)$ satisfies
\begin{equation*}
\mathrm{sign}\bigl( \beta_{\psi^\star}(s) - \beta_0 \bigr)
\;=\;
\mathrm{sign}\!\left(\, \mathbb{E}\!\left[\, I_t^{+,m}\,\hat A_t^{E} \,\big|\, s_t = s \right] \right)
\end{equation*}
at the unconstrained interior minimum, and is clipped to $\beta_\text{min}$ or $\beta_\text{max}$ on the boundary. Equivalently, $\beta_{\psi^\star}$ is monotone in the conditional cross-moment between $I^{+,m}$ and $\hat A^{E}$.
Proof in Appendix~\ref{app:proof_target}.
\end{proposition}

Proposition~\ref{prop:target_recovery} says that, in the population limit and ignoring regularization, the optimal Beta Network raises intrinsic reward (outputs a value above $\beta_0$) at states where the intrinsic signal $I^{+,m}$ tends to move together with positive extrinsic advantage, and lowers it otherwise. The next result describes the gradient direction during training.

\begin{proposition}[Gradient Direction]
\label{prop:gradient_direction}
Treat $\beta_\psi(s)$ as a free parameter at a state $s \in \mathcal{S}$ visited with positive probability. Then
\begin{equation*}
\frac{\partial L_\text{corr}}{\partial \beta_\psi(s)}
\;=\;
-\,\frac{\mathbb{P}(s_t = s)}{\sigma_{\tilde I}}\,
\Bigl(
\mathbb{E}[\, I_t^{+,m}\,\hat A_t^{z} \mid s_t = s \,]
\;-\;
\mathbb{E}[\, I_t^{+,m} \mid s_t = s\,]\, \mathbb{E}[\, \hat A_t^{z} \,]
\Bigr).
\end{equation*}
In particular, gradient descent on $L_\text{corr}$ increases $\beta_\psi(s)$ when $I^{+,m}$ and $\hat A^{E}$ are conditionally positively correlated at $s$, and decreases it when they are conditionally negatively correlated.
Proof in Appendix~\ref{app:proof_gradient}.
\end{proposition}

Proposition~\ref{prop:gradient_direction} makes the main design idea of CARE clear. A gradient step pushes $\beta_\psi(s)$ in the direction that would increase the within-batch correlation between weighted curiosity and extrinsic advantage at state $s$. The Beta Network is therefore supervised by the agreement between the intrinsic generator and the extrinsic critic. No extra network beyond the critic itself is needed.

\subsection{First-Order Optimization vs. Meta-Gradient}
\label{sec:first_order}

A natural alternative to the correlation objective is to learn the intrinsic reward weighting via meta-gradients, as in Learning Intrinsic Rewards for Policy Gradient methods (LIRPG) \cite{zheng2018lirpg}. In that approach, the parameters of the intrinsic reward function are updated so that, after a policy update using the shaped reward, the new policy maximizes the extrinsic objective. This requires differentiating through one or more inner policy updates. The differentiation is costly and can be unstable in deep reinforcement learning settings.

CARE replaces the meta-gradient by a first-order objective. This objective supervises $\beta_\psi$ directly through the within-batch correlation between weighted intrinsic reward and extrinsic GAE advantage. The cost difference is given by the following result.

\begin{proposition}[Computational Cost]
\label{prop:cost}
Per Beta-network update of size $T$, let $K$ denote the number of inner policy update steps used in a meta-gradient estimator, $|\theta|$ the number of policy parameters, and $|\psi|$ the number of Beta-network parameters. Then a meta-gradient training step has worst-case time complexity $O(K\, T\, |\theta|)$, while a CARE first-order training step has time complexity $O(T\, |\psi|)$. In particular, when $|\psi| \ll |\theta|$ and $K \ge 1$, the CARE update is asymptotically cheaper by a factor of $K\, |\theta| / |\psi|$.
Proof in Appendix~\ref{app:proof_cost}.
\end{proposition}

In the implemented system, $|\psi|$ is about two orders of magnitude smaller than $|\theta|$. So the cost saving is large. The price of this efficiency is a weaker form of supervision. Instead of optimizing $\beta_\psi$ directly for future extrinsic performance, CARE optimizes it for the within-batch correlation between weighted intrinsic curiosity and extrinsic advantage. This correlation is a proxy for that performance. The next result describes two cases in which the proxy reduces to the simple fixed-coefficient case.

\begin{corollary}[Degeneration to Fixed Coefficient]
\label{cor:degeneration}
Consider the objective $L_\beta(\psi)$ in Equation~\eqref{eq:beta_loss}. There are two regimes in which every minimizer $\psi^\star$ satisfies $\beta_{\psi^\star}(s) \to \beta_0$ uniformly in $s$, and the shaped reward in Equation~\eqref{eq:combined_reward} reduces to the fixed-coefficient form $\bar r_t = R_t^E + \beta_0\, I_t^{+,m}$:
\begin{enumerate}
    \item[(i)] \textbf{Cold-start regime}: when the rollout buffer contains no informative extrinsic signal, so that $\hat A_t^{E} = 0$ for almost every $t$, the standardization in Equation~\eqref{eq:zscored_signals} is undefined and the correlation gradient vanishes. In this regime $\nabla_\psi L_\text{corr} = 0$ identically, and the only active term in $L_\beta$ is the regularizer $L_\text{reg}$, whose unique minimizer is $\beta_\psi \equiv \beta_0$.
    \item[(ii)] \textbf{Strong regularization regime}: as $\lambda_\text{reg} \to \infty$, the regularization term dominates and forces $\log \beta_\psi(s) \to \log \beta_0$ in $L^2$ under the state-visitation distribution, equivalently $\beta_\psi(s) \to \beta_0$.
\end{enumerate}
Proof in Appendix~\ref{app:proof_degeneration}.
\end{corollary}

Corollary~\ref{cor:degeneration} has both a theoretical and a practical reading. In theory, the corollary shows that CARE contains the fixed-coefficient baseline PPO + module $m$ (with coefficient $\beta_0$) as a limit case for any intrinsic reward module $m$. So the framework cannot do worse than that baseline once the regularization is well tuned. In practice, the cold-start branch describes the behavior of CARE in very sparse environments. When no extrinsic reward has been seen yet, the GAE advantage carries no signal-strength information across states. The correlation gradient is then roughly zero-mean noise. The regularizer alone keeps $\beta_\psi$ anchored at $\beta_0$ until extrinsic signal appears.

\section{Algorithm}
\label{sec:algorithm}

This section describes the data structures, loss functions, and training procedure used to optimize the system end to end.

\subsection{Rollout Buffer}
\label{sec:rollout_buffer}

Each PPO update operates on an on-policy rollout buffer of length $T$ collected by the current policy. A single transition is the tuple
\begin{equation}
\xi_t \;=\; \bigl( s_t,\; a_t,\; \log \pi_\theta(a_t \mid s_t),\; R_t^E,\; s_{t+1},\; d_t \bigr),
\label{eq:transition_tuple}
\end{equation}
where $d_t \in \{0, 1\}$ is the episode termination flag. The buffer of length $T$ is stored as the tensors
\begin{equation*}
\mathcal{S}_\text{tensor} \in \mathbb{R}^{T \times d_s}, \quad
\mathcal{A}_\text{tensor} \in \mathbb{N}^{T}, \quad
\mathcal{R}_\text{tensor} \in \mathbb{R}^{T}, \quad
\mathcal{S}'_\text{tensor} \in \mathbb{R}^{T \times d_s}, \quad
\mathcal{D}_\text{tensor} \in \{0,1\}^{T},
\end{equation*}
together with the log-probabilities $\mathcal{P}_\text{tensor} \in \mathbb{R}^{T}$. Following the implementation defaults, $T = 4 \times \text{max\_ep\_len}$ steps are collected before each PPO update.

\subsection{Loss Functions}
\label{sec:loss_functions}

Four separate losses are minimized at each update cycle. The losses share the same rollout buffer but are optimized over different parameter groups. So no joint optimization is needed.

\textbf{PPO Actor Loss $L^\text{PPO}(\theta)$.} Define the importance ratio $\rho_t(\theta) = \pi_\theta(a_t \mid s_t) / \pi_{\theta_\text{old}}(a_t \mid s_t)$ and let $\hat A_t$ denote the advantage estimate computed by Generalized Advantage Estimation (GAE) \cite{schulman2016gae} from the shaped reward $\bar r_t$. The clipped surrogate objective \cite{schulman2017ppo} is
\begin{equation}
L^\text{PPO}(\theta) \;=\;
-\,\mathbb{E}_t\!\left[
\min\!\Bigl(
\rho_t(\theta)\, \hat A_t,\,
\text{clip}\bigl(\rho_t(\theta),\, 1-\epsilon,\, 1+\epsilon\bigr)\, \hat A_t
\Bigr)\right].
\label{eq:ppo_loss}
\end{equation}

\textbf{Critic Loss $L^V(\mu)$.} The value network $V_\mu$ is fitted by mean-squared error against the GAE return target $\hat V_t$:
\begin{equation}
L^V(\mu) \;=\; \mathbb{E}_t\!\left[ \bigl( V_\mu(s_t) - \hat V_t \bigr)^2 \right].
\label{eq:critic_loss}
\end{equation}

\textbf{ICM Loss $L_\text{ICM}(\eta)$.} As specified in Equation~\eqref{eq:icm_loss}.

\textbf{Beta Loss $L_\beta(\psi)$.} As specified in Equation~\eqref{eq:beta_loss}.

The four losses match four parameter groups: $\theta$ (actor), $\mu$ (critic), $\eta$ (ICM), and $\psi$ (Beta Network). Gradients flow through each loss only with respect to its own parameter group. So the Beta Network is not updated through $L^\text{PPO}$, and the policy is not updated through $L_\beta$.

\textbf{Live Beta Network in shaped reward.} Following \cite{andres2025impact}, the shaped reward in Equation~\eqref{eq:combined_reward} uses the \emph{live} Beta Network $\beta_\psi$ rather than a Polyak-averaged target. The circular dependency within one update, which a target network would normally avoid, is broken instead by the temporal ordering of the four loss updates. The extrinsic GAE advantage $\hat A_t^{E}$ used in $L_\beta$ is computed from the critic $V_\mu$ at its values \emph{before} the current PPO update. So $\hat A_t^{E}$ is fixed when $\psi$ is updated, and is not affected by the in-progress actor or critic step. A single $\psi$-step is taken per outer PPO iteration, in line with Algorithm 1 of \cite{andres2025impact}.

\subsection{Training Procedure}
\label{sec:training_procedure}

Algorithm~\ref{alg:care} specifies the complete CARE training loop. One iteration of the outer loop corresponds to a single PPO update cycle.

\begin{algorithm}[H]
\caption{CARE training loop with PPO and an intrinsic reward module.}
\label{alg:care}
\begin{algorithmic}[1]
\State Initialize policy $\theta$, critic $\mu$, intrinsic-module parameters $\eta$ (if any), Beta Network $\psi$
\For{$\text{update} = 1, 2, \dots, U$}
    \State Collect $T$ on-policy transitions $\{\xi_t\}_{t=0}^{T-1}$ using $\pi_\theta$
    \State Compute raw intrinsic rewards $I_t$ via the module-specific equation in Section~\ref{sec:intrinsic_modules}
    \State Standardize and rectify to obtain $I_t^+$ via Equation~\eqref{eq:icm_standardized}
    \State Update intrinsic-module parameters $\eta$ (no-op if module is non-parametric, e.g., count-based)
    \State Compute extrinsic GAE advantages $\hat A_t^{E}$ via Equation~\eqref{eq:gae_advantage_ext} from the current critic $V_\mu$ (no rollout-level normalization)
    \State Update Beta Network with one gradient step: $\psi \gets \psi - \psi_\text{lr}\, \nabla_\psi L_\beta(\psi;\, \{s_t, I_t^+, \hat A_t^{E}\})$, with $L_\beta$ defined in Equation~\eqref{eq:beta_loss}
    \State Compute shaped rewards $\bar r_t = R_t^E + \beta_\psi(s_t)\, I_t^+$ using the updated $\psi$
    \State Compute GAE advantages $\hat A_t$ from $\{\bar r_t\}$ and $V_\mu$ (rollout-level normalization for PPO)
    \For{$\text{epoch} = 1, 2, \dots, K$}
        \State Update actor: $\theta \gets \theta - \theta_\text{lr}\, \nabla_\theta L^\text{PPO}(\theta)$
        \State Update critic: $\mu \gets \mu - \mu_\text{lr}\, \nabla_\mu L^V(\mu)$
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

The algorithm keeps the standard PPO update structure (lines 11--14) and adds only two extra optimization steps (lines 6 and 8) before the shaped reward is computed. The extrinsic-only advantage $\hat A_t^{E}$ used in line 7 is computed from the same critic $V_\mu$ that drives the PPO update. But rollout-level normalization is disabled so that the cross-state size is kept. The PPO advantage $\hat A_t$ in line 10 is computed again from the shaped reward, and is normalized in the standard way. So the two advantage computations differ only in their reward stream and in whether normalization is applied. This modularity is important. The same template works if ICM is replaced by a count-based bonus or by RIDE. Only Equations~\eqref{eq:icm_raw}--\eqref{eq:icm_standardized} are swapped for the corresponding module-specific computation.
